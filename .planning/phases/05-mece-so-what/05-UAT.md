# Phase 5 UAT — MECE + So-what 行为验证脚本

**Purpose:** 对 idea-to-task skill 的 Phase 5 完成态做一次性行为 spot-check，验证 `decision-rules.md` 第 6 节与第 7 节以及 `SKILL.md` 第 6 步是否真正生效。本脚本只作为人工 UAT 手册，不新增 fixture 文件。

**执行方式：**
1. 将每类「测试输入」原文单独喂给 idea-to-task skill。
2. 收到输出后，按对应的「行为断言清单」逐条核对。
3. 记录通过 / 失败 / 部分通过，并将结论写入 `05-VERIFICATION.md`。

**术语黑名单**（所有类别共用，任一命中即整类失败）：
- SCQA / Situation / Complication / Question / Answer
- 支柱 / 金字塔 / pillar / MECE / So-what
- mece_check / sowhat_filter / grouping / inductive / deductive / 分组

---

## 类别 1：战略型输入

### 输入
```text
[在此填入一段跨部门协同 + 治理节奏信号明确的 CEO 碎片原文，200-400 字。示例主题：「公司级数字化转型推进」或「多事业部 OKR 对齐」。必须包含跨部门、资源竞争、治理节奏等信号词]
```

### 行为断言清单

- [ ] 核心判断段出现「先做总体方案」或「总体方案设计」相关表述
- [ ] 主题 I 的名称以「总体方案设计」结尾或等价措辞
- [ ] 主题 I 的里程碑至少包含目标与边界、现状诊断、治理机制、推进路线中的 3 项
- [ ] 主题数量落在 2-4 之间
- [ ] 同层主题分组标准一致，不混用「按业务线」和「按时间阶段」
- [ ] 无任一术语黑名单命中

### 失败判定

- 主题 I 跳到工具选型、部门动作或预算项，判定为 MECE §6 分组一致性失败
- 出现「MECE」「支柱」等内部术语，判定为术语泄漏

---

## 类别 2：归纳型输入

### 输入
```text
[填入一段同时提到 3-4 个看似独立但本质服务同一顶层目标的方向。示例：「加强产品力 + 提升研发效率 + 优化组织协同」最终均服务于「打造下一代旗舰产品」]
```

### 行为断言清单

- [ ] 第 2 段「主题拆分」标注「单主题（归纳）」
- [ ] 核心判断段说明归纳来源，明确原始拆出的方向共同服务于同一顶层目标
- [ ] 原始 3-4 个方向降级为主题下的里程碑，不再作为并列主题
- [ ] 里程碑数量落在 2-4 之间
- [ ] 无任一术语黑名单命中

### 失败判定

- 输出并列的 3-4 个主题，判定为归纳聚合失败
- 缺少「单主题（归纳）」标记，判定为 QUAL-01 完备性失败

---

## 类别 3：常规型输入

### 输入
```text
[填入一段包含 2-4 个天然独立方向的碎片。示例：「H2 要做 3 件事：重构核心交易链路 / 搭建数据中台 / 准备区域扩张」]
```

### 行为断言清单

- [ ] 输出 2-4 个并列主题，不强拼「总体方案」第一主题
- [ ] 同层分支互斥，不出现「提升效率 vs 优化流程」这类重叠
- [ ] 每个主题下至少 1 条可指派 + 可验收的里程碑
- [ ] 无 5 类空洞模式命中，手动检查每一条任务描述
- [ ] 无任一术语黑名单命中

### 失败判定

- 强拼战略型结构，判定为战略型触发条件误判
- 出现「加强 X」「推动 Y 落地」等空洞项，判定为 §7 过滤失败

---

## 类别 4：中等证据 3 层输入

### 输入
```text
[填入一段背景 + 矛盾清楚但核心判断需要推演的输入，约 150-300 字。应支持展开到里程碑层，但不应硬造执行项]
```

### 行为断言清单

- [ ] 任务树展开到里程碑层（3 层结构：核心判断 → 主题 → 里程碑）
- [ ] 不强造第四层执行项；若出现执行项，也只应出现在个别信息显著更充分的主题下
- [ ] 每层分支数均落在 2-4 之间
- [ ] 里程碑描述均包含可验收要素，如时点、交付物或数量
- [ ] 无任一术语黑名单命中

### 失败判定

- 强造执行项层，判定为弹性层级规则失败
- 里程碑全是「持续 X」「加强 Y」等空洞表述，判定为 §7 可验收性失败

---

## 类别 5：极弱证据 2 层输入

### 输入
```text
[填入一句极短的模糊方向口号，例：「要重视 AI」或「把用户体验做好」，不超过 30 字]
```

### 行为断言清单

- [ ] 先触发 2-3 个定向追问，或直接输出到主题层停止
- [ ] 若选择直接输出，任务树停在主题层（2 层结构），不造第三层
- [ ] 第 5 段「风险与高风险推断」必须标注「输入不足以支撑多主题拆分」或等价的层级受限表述
- [ ] 不伪造具体执行项
- [ ] 无任一术语黑名单命中

### 失败判定

- 硬造里程碑层或执行项层，判定为 §3 弹性层级 + §6 underflow 兜底失败
- 风险段无层级受限标注，判定为 D-06 失败
