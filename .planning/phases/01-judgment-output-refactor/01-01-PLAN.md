新增节内容如下（per D-01, D-02, D-03, D-08）：

```
## 3. 向上归纳检查

**在主题密度判断拆出多个主题后执行。目标是检测这些主题之间是否存在目标从属关系，如果存在则收敛为单顶层主题。**

适用条件：
- 主题密度判断结果为「多相关主题」或「多不相关主题」时触发
- 不限于战略型输入 — 所有多主题输入都先尝试归纳（per D-01）

归纳判断标准（per D-02）：
- 当多个主题服务于同一个更高层目标时，判定存在目标从属关系
- 判断方式：逐一检查每个主题，问「去掉这个主题后，其他主题还能独立成立吗？」
  - 如果去掉某个主题后其他主题失去方向或上下文 → 该主题是顶层目标
  - 如果每个主题都能独立推进、互不影响 → 不存在从属关系

归纳成功时：
- 将从属主题收敛为单顶层主题，原来的并列主题降级为该顶层主题下的子任务线
- 输出走「归纳型」模板（见 output-template.md 归纳型输出部分）

归纳失败时（per D-08）：
- 多个主题确实独立、无法归纳 → 静默回退走现有的平铺多主题逻辑
- 不向读者暴露「尝试归纳但失败」的内部过程（符合「不暴露内部推理」原则）
```

同时更新后续所有节编号：

**2. 修改「## 2. 主题拆分」部分（per D-06, OUT-02）：**

在现有内容之后追加归纳型主题拆分格式说明：

```
**归纳型输出时**，主题拆分标记为「单主题（归纳）」，说明是由多个方向收敛而来，区别于天然单主题：

- 单主题（归纳）：原始输入包含多个方向，经向上归纳检查确认存在目标从属关系，收敛为单顶层主题
- 单主题：输入本身就只围绕一个明确方向
```

````
归纳型与战略型的区别：
- 战略型：第一主题固定为「总体方案设计」，后续分诊断线和落地线
````
