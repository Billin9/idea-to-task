# 标准输出模板

## 1. 核心判断

用 3 到 6 句话回答：

- CEO 真正在推动什么
- 主线是什么
- 次线是什么
- 当前更偏战略、产品还是执行

## 2. 主题拆分

如果输入包含多个方向，先按主题拆开。
如果本来就是单主题，明确写“单主题”即可，不要把里程碑、时限、试用验证拆成主题。

## 3. 分层任务树

按主题输出：

- 项目级
- 里程碑级
- 执行项级

如果证据不足，允许缺省执行项级，但必须说清楚为什么停在更高层。

## 4. 执行建议

至少包含：

- 优先级
- 建议负责人角色
- 推荐推进顺序

## 5. 风险与高风险推断

只提醒真正会改变任务方向的推断。
如果存在两条相反路径，必须明确写出这是高风险歧义。

## 6. 下一步动作

固定给出 3 到 5 条最值得先推进的动作。

## 输出风格

- 优先短句
- 不写成长篇纪要
- 不重复原话
- 让团队看完就知道下一步怎么动
- 不输出 `<thinking>`、分析过程或元说明
