# 标准输出模板

## 1. 核心判断

用 3 到 6 句话回答：

- CEO 真正在推动什么
- 主线是什么
- 次线是什么
- 当前更偏战略、产品还是执行

**归纳型核心判断**（当向上归纳检查成功时使用，仍控制在 3 到 6 句内）：

- 第一句写顶层意图：「CEO 真正在推动的是：[顶层意图]。」
- 第二句说明归纳来源：「原始拆出的 [主题A]、[主题B]、[主题C] 均服务于这一顶层目标，归纳为单主题。」
- 后续句描述当前阶段重心和主要挑战

## 2. 主题拆分

如果输入包含多个方向，先按主题拆开。
如果本来就是单主题，明确写”单主题”即可，不要把里程碑、时限、试用验证拆成主题。

如果经过向上归纳检查后收敛为单主题，标记为「单主题（归纳）」，说明是由多个方向收敛而来，区别于天然单主题。

## 3. 分层任务树

**模板选择优先级（从高到低）：**

1. **战略型模板**：输入已触发战略型判断（见 decision-rules.md 第 1 节）→ 强制使用战略型模板
2. **归纳型模板**：多主题输入经向上归纳检查成功（见 decision-rules.md 第 3 节）→ 使用归纳型模板
3. **常规模板**：其他所有情况（单主题、多主题归纳失败）→ 按主题输出

优先级不可跨级：已触发战略型的输入不走归纳型模板，归纳成功的输入不走常规多主题模板。

**战略型输入（公司级变革）**，第一主题固定结构如下（标题按场景替换，结构保持一致）：

```
### 主题 I：[场景名称] 总体方案设计
*建立全局蓝图，明确目标、边界、治理规范与推进节奏。落地类主题的执行依赖此主题先行完成。*

- **里程碑（核心必含）**
  - [ ] 定义目标与边界
  - [ ] 现状诊断与约束盘点
  - [ ] 划分各主线推进路径
  - [ ] 建立治理机制
  - [ ] 绘制分阶段推进路线图
- **里程碑（按场景选配）**
  - [ ] 预算原则与资源编排
  - [ ] 组织分工与责任矩阵
  - [ ] 试点选择与验收标准
  - [ ] 能力建设与培训计划
  - [ ] 度量体系与考核指标
- **具体执行项**
  - 明确第一阶段进入范围（部门/系统/场景）
  - 输出《XX 总体方案大纲》
```

之后的主题分为两类：
- **可并行的诊断线**（现状盘点、工具摸底、试点候选识别等，可与总体方案同步推进）
- **必须后置的落地线**（部门改造、预算编排、规模化推广等，等总体方案确定后推进）

**归纳型输入（多主题收敛为单顶层意图）**，顶层主题为归纳后的统一目标，原各主题降级为里程碑：

```
### 主题：[顶层意图名称]
*由 [原主题A]、[原主题B]、[原主题C] 归纳而来。*

- **里程碑 1：[原主题A 方向]**
  - [ ] [执行项]
  - [ ] [执行项]
- **里程碑 2：[原主题B 方向]**
  - [ ] [执行项]
- **里程碑 3：[原主题C 方向]**
  - [ ] [执行项]
- **具体执行项**
  - 明确第一阶段聚焦的里程碑
  - 输出整体推进节奏
```

里程碑的排列顺序按依赖关系和优先级排序，不按原始提及顺序。

**非战略型输入**，按主题输出：

- 项目级
- 里程碑级
- 执行项级

如果证据不足，允许缺省执行项级，但必须说清楚为什么停在更高层。

## 4. 执行建议

至少包含：

- 优先级
- 建议负责人角色
- 推荐推进顺序

## 5. 风险与高风险推断

只提醒真正会改变任务方向的推断。
如果存在两条相反路径，必须明确写出这是高风险歧义。

## 6. 下一步动作

固定给出 3 到 5 条最值得先推进的动作。

## 输出风格

- 优先短句
- 不写成长篇纪要
- 不重复原话
- 让团队看完就知道下一步怎么动
- 不输出 `<thinking>`、分析过程或元说明
