# SCQA 推演与风险边界

## 1. SCQA 推演

**所有输入统一走这一节。** 不再先做战略型、归纳型或常规型分流；先在内部完成一轮有序推演，再由推演结果自然导出后续处理方式。

当你处理输入时，按下面顺序在内部完成推演：

1. **提取背景**
   - 当输入直接给出业务现状、已有动作、组织范围或时间背景时，优先采用这些明确信息。
   - 当输入没有说全，但上下文已经暗示了前提、目标对象或约束时，可以补出最小必要背景，不要为了凑完整而扩写。
2. **提取矛盾**
   - 当输入明确提到痛点、变化压力、时点要求、资源限制或推进阻力时，把这些视为打破现状的关键矛盾。
   - 当表面上只有方向口号，但语气里已经包含“必须现在做”“不能再拖”“多线协同会失控”等压力信号时，也要识别为矛盾线索。
3. **推导核心问题**
   - 核心问题必须从“背景”和“矛盾”的张力里自然长出来，回答“现在到底要解决什么”。
   - 当现有张力无法支撑一个稳定的问题时，不要硬造问题，转去第 2 节判断是否需要追问。
4. **推导核心判断**
   - 核心判断必须直接回应上一步的问题，给出可行动的方向，而不是“先分析一下”“先看看再说”这类回避表述。
  - 当多个主题本质上都在服务同一个核心判断时，直接收敛为一个顶层方向；原来的多主题只作为后续里程碑或子任务线展开，不再单独设置归纳步骤。

内部推理统一使用以下格式，且只允许存在于内部：

```xml
<scqa_analysis>
S: [从输入中提取的背景]
C: [从输入中提取的矛盾]
Q: [从张力中推导出的核心问题]
A: [对核心问题的方向判断]
完整度: [强/中/弱]
</scqa_analysis>
```

明确限制：

- 用户可见输出只呈现自然语言的“核心判断”，不出现任何方法论术语，也不暴露内部推理过程。
- 不把推演写成并行清单；必须先看背景，再看矛盾，再推问题，最后落到方向判断。

### 特殊输入类型处理

当推演结果显示这是一个跨多条业务线、需要统一目标和治理节奏、且周期跨季度的公司级推动事项时，核心判断必须保留“先做总体方案设计”的约束：

- 第一主题先落在总体方案，而不是直接跳到工具、部门动作或预算项。
- 总体方案至少覆盖目标与边界、现状与约束、治理机制、推进路线。
- 其余主题再按“可并行的诊断线”和“必须后置的落地线”展开。

## 2. SCQA 追问触发

只有当关键要素缺失并且会改变任务方向时，才触发定向追问。SCQA 定向追问最多 2 到 3 个，与后续通用补问分开计数。

当出现以下情况时触发追问：

- **只有模糊背景，没有关键矛盾**：例如只有愿景口号或泛泛方向，无法判断当前挑战是什么。此时追问应聚焦“现在卡在哪里”或“为什么此刻要做”。
- **背景和矛盾都在，但可能导向两条相反方向**：例如同样的现状既可能导向“先平台化”，也可能导向“先单点验证”。此时追问应聚焦决策倾向，而不是补背景细节。
- **输入过短，四项信息都提不起来**：例如只有一句口号、一个名词或一个模糊目标。此时追问应补背景、约束和成功标准。

当出现以下情况时不追问：

- 背景和矛盾已经足以推出稳定方向，即使把握不是百分之百。
- 输入不完美，但主线已经清楚，只差执行层细节。

追问表达要求：

- 每个问题都要说明“这个答案会如何改变任务方向”。
- 问题必须短、具体、能直接影响主题划分、优先级或推进路径。

## 3. 证据强度与 SCQA 完整度

证据强度由内部推演完整度决定，不再依赖旧的前置分流。

- **强证据**：四项信息齐全，背景、矛盾、核心问题、核心判断都能稳定成立，且方向可以直接落到行动层。
- **中等证据**：背景和矛盾明确，核心问题与核心判断需要结合上下文推演，但推演路径基本唯一。
- **弱证据**：只有模糊背景，关键矛盾不清楚，或同一输入还能合理导出多条相反方向。

当前阶段只建立“完整度 → 证据强度”的映射，具体展开层级会在 Phase 4 用金字塔弹性结构替代。为了兼容当前模板，暂时保留以下深度接口：

- 弱证据：只输出项目级
- 中等证据：可输出到里程碑级
- 强证据：才允许输出执行项级

## 4. 高风险歧义

出现以下场景时，优先双路径输出：

- “先做高端还是先做大众”
- “先做平台还是先做单点功能”
- “现在投入还是继续观察”

## 5. 常见基线偏差

没有专用 skill 时，模型常出现这些偏差：

- 只做摘要，不做任务树
- 会提一些问题，但不给推进顺序和角色建议
- 看到模糊内容就平均分配篇幅，没把主线压出来
- 能看出两条路线，但不会显式标记高风险歧义

你的目标是修正这些偏差，而不是把内容写得更长。
